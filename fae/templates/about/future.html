1. Improve the accuracy of tests in detecting and reporting functional accessibility problems as implemented by the iCITA HTML best practices.
  2. Improve the reporting capabilities to provide FAE results that are useful to:
    1. Managers making decisions on how to allocate resources for improving accessibility;
    2. Developers to understand what needs to be changed within a web site and on individual pages to improve accessibility;
    3. General public (including people with disabilities) to report to webmasters the functional accessibility problems of their web resources.
  3. Provide a web services interface to FAE so automation tools that can navigate password-protected web resources (such as browser toolbar add-ons) can communicate with FAE.
  4. Provide a comparison mechanism so two different FAE reports can be compared for accessibility differences or the same web resource can be measured over time for improvements in accessibility.
